# Step-back Prompt
在设计定价方案前，先思考以下底层问题：
1. 用户为该产品付费的核心驱动力是什么（节省时间/提升效率/获取独占价值）？
2. 用户心智中的价格锚点是什么（竞品价格/替代方案成本/自建成本）？
3. 不同用户群体的付费能力和意愿差异有多大？

# Few-shot Example
> 示例：某AI写作工具的套餐设计摘要
> - 免费版：每日3次AI生成、基础模板，目的：降低试用门槛，积累用户池
> - 基础版（¥29/月）：每日50次AI生成、全部模板、导出PDF，目标用户：轻度个人用户
> - 专业版（¥79/月）：无限AI生成、高级改写、团队协作、API接入，目标用户：内容创作者/自媒体
> - 企业版（¥299/席位/月）：私有化部署、专属模型微调、SLA保障、管理后台，目标用户：企业内容团队
> - 单位经济：CAC ¥45 → LTV ¥680（按14个月平均留存）→ LTV/CAC = 15.1 → 回收期2.1个月

# Constraints
- 套餐权益边界须精确清晰，相邻层级有明确价值阶梯
- LTV/CAC须大于3，回收期控制在合理范围内
- 免费版提供足够价值建立使用习惯，同时预留清晰的付费升级动机
- 定价在功能稳定且有用户验证数据后最终确定
- 所有价格须经过竞品对标和用户调研验证
- 企业版定价须预留销售谈判空间
