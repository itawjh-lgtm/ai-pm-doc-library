## 二、认证授权流程

### 2.1 认证方式说明
- 认证协议：[JWT / OAuth2.0 / API Key]
- Token获取方式：[登录接口 / 授权码模式 / 客户端凭证模式]
- Token有效期：[Access Token X分钟 / Refresh Token X天]
- Token刷新机制：[自动续期 / 手动刷新]

### 2.2 认证流程图
```
[客户端] → POST /auth/login (账号密码) → [服务端]
[服务端] → 返回 access_token + refresh_token → [客户端]
[客户端] → 携带 Authorization: Bearer {access_token} → [服务端]
[服务端] → Token过期返回401 → [客户端]
[客户端] → POST /auth/refresh (refresh_token) → [服务端]
```

### 2.3 请求头格式
```json
{
  "Content-Type": "application/json",
  "Authorization": "Bearer {access_token}",
  "X-Request-ID": "{唯一请求标识}",
  "X-Client-Version": "{客户端版本号}"
}
```

## 五、通用规范

### 响应格式
```json
{
  "code": 200,
  "message": "success",
  "data": {},
  "request_id": "uuid-xxxx",
  "timestamp": 1234567890
}
```

### 通用错误码
| 错误码 | 含义 | 处理建议 |
|--------|------|---------|
| 400 | 请求参数错误 | 检查入参格式和必填项 |
| 401 | 未认证/Token过期 | 刷新Token或重新登录 |
| 403 | 无权限 | 检查角色权限/联系管理员 |
| 404 | 资源不存在 | 检查资源ID是否正确 |
| 409 | 资源冲突 | 检查幂等性/重试 |
| 422 | 参数校验失败 | 检查业务规则约束 |
| 429 | 请求过频 | 降低频率/查看X-RateLimit-Reset |
| 500 | 服务器内部错误 | 记录request_id联系开发排查 |
| 502 | 网关错误 | 稍后重试 |
| 503 | 服务不可用 | 稍后重试/检查服务状态 |
